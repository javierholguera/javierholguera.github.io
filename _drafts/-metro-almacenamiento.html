En las Aplicaciones Metro con HTML/Javascript podemos tener varios tipos de almacenamiento:

  • HTML5 Web Storage
  • File API
  • Indexed Database API
  • Application Cache API

Veamos qué uso tiene cada uno de estos mecanismos.

HTML5 Web Storage

Este sencillo estándar sigue una estructura parecida a las cookies, basada en parejas clave/valor. Sin embargo, tiene algunas ventanas como:

  • No se transmite la información en cada petición, a diferencia de lo que ocurren con las cookies.
  • Soporta compartir la información a través de múltiples ventanas (sessionStorage)
  • Soporta compartir la información más allá de la sesión actual (localStorage)

La página de su estándar en W3C aquí.

File API

Este estándar está orientado a la lectura y manipulación de ficheros. Se basa en una serie de interfaces que realizan distintas funciones:

  • FileList: representa una lista de ficheros.
  • Blob: representa información binaria inmutable.
  • File: representa los atributos de sólo lectura de un fichero.
  • FileReader: provee métodos para leer un File o un Blob.

Este estándar es más reciente e inmaduro que, por ejemplo, WebStorage, y su soporte es parcial en casi todos los navegadores. La página de su estándar en W3C aquí.

Indexed Database API

Este estándar representa una base de datos capaz de almacenar tanto valores simples como jerarquías de objetos. La principal característica de este estándar es que está totalmente desvinculado de SQL, precisamente uno de sus requisitos desde sus primeras etapas.

Tiene APIs tanto síncrona como asíncrona (para utilizar con Web Workers) y su uso es preferible frente a alternativas como WebStorage cuando se va a almacenar una gran cantidad de información o existe una cierta complejidad en sus relaciones, puesto que es mucho más potente.

La página de su éstandar en W3C aquí.

Application Cache API

Esta API es parte del estándar HTML5 y está enfocada a facilitar el uso offline de las aplicaciones web, además de mejorar su rendimiento permitiendo registar objetos que se quieran cachear y que estén disponibles en un entorno desconectado.

La sección correspondiente en el estándar HTMl5 en W3C aquí.
